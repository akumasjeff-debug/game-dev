# 台灣熱炒王 — 教學關卡設計

版本：v1.0
設計組：教學設計師
日期：2026-06-20

---

## 概覽

### 教學目標

讓玩家在前 5 分鐘「懂了、會了、想繼續玩」。

三天教學各有主題，難度遞進：
- Day 1 全程手把手，引導玩家完成第一次完整服務循環
- Day 2 半引導，玩家主動操作建造模式，系統提示但不鎖操作
- Day 3 只給提示不鎖手，玩家自行探索雇用與菜單管理

### 設計哲學

1. 動手做比看說明有效：每一步都是玩家的親手操作，無法純靠看過
2. 每步只教一件事：不讓玩家同時記兩個新概念
3. 第一分鐘就有成就感：Day 1 步驟 1 的目標是「讓第一位客人坐下」，60 秒內完成
4. 教學給資源優惠：Day 1 客人數量固定（10 組）且耐心值加倍，降低失敗機率
5. 組合加成在 Day 1 就露一次臉：讓玩家早早見識到「搭配有加成」的爽感
6. 失敗也是教學：客人等太久離席時，系統即時說明原因，不懲罰名聲

### 教學期間特殊規則（程式實作用）

- Day 1-3 名聲不會因失誤扣分（只加不減，容錯保護）
- Day 1 強制客流：固定 10 組老街坊，間隔穩定，不受名聲值影響
- Day 2 建造模式解鎖：首次進入施工模式前，格子系統鎖定不可互動
- Day 3 結束後移除所有教學限制，遊戲進入正式流程

## Day 1：第一鍋炒起來

### 概述

主題：認識基本操作。
教學目標：讓玩家完成一次完整的服務流程（客人進門 → 點餐 → 炒菜 → 送餐 → 收款）。

開場設定：
- 遊戲日 17:00，開店前一刻
- 畫面切入店內俯視圖，阿龍師傅站在快炒爐旁，玩家角色（老闆娘）在收銀台前
- 初始場地已預置：1 個快炒爐（廚房區左上）、1 張方桌（外場區中央）、1 條走道（縱向穿越中央）

---

### 步驟 1：認識你的店

- 觸發條件：遊戲首次啟動，載入完成後自動觸發
- 完成條件：玩家點擊螢幕任意位置（確認看過介紹）
- 對話提示：
  - [阿龍師傅]「妳就是新老闆娘？好，我先帶妳看一下我們這間店。」
  - [系統]「廚房區（橘）：炒菜的地方。外場區（米黃）：客人坐的地方。走道（灰）：送餐通道。」
- UI 高亮：整個地圖，三個區域各以對應顏色閃爍 1.5 秒
- 跳過條件：無（首次必看）

---

### 步驟 2：開門！第一組客人來了

- 觸發條件：步驟 1 完成後，自動進入 17:00 開店時段
- 完成條件：玩家點擊螢幕下方中央的「開門」按鈕
- 對話提示：
  - [老闆娘]「好，時間到了，開門開門！」
  - [系統]「點擊下方的『開門』按鈕開始營業。」
- UI 高亮：螢幕下方中央的「開門」按鈕，橘色邊框閃爍
- 跳過條件：無

---

### 步驟 3：帶客入座

- 觸發條件：開門後，第一組客人（2 人老街坊）出現在入口
- 完成條件：客人走到方桌並坐下（AI 自動執行，玩家觀看即可）
- 對話提示：
  - [阿龍師傅]「來了來了，老街坊。他們會自己找桌子坐，妳看著。」
  - [系統]「客人進門後會自動尋找空桌入座。桌子旁邊必須有走道，客人才過得去。」
- UI 高亮：從入口到方桌的走道格，以白色箭頭動態指引路徑
- 跳過條件：無

---

### 步驟 4：接單！看訂單出現

- 觸發條件：客人坐下後，訂單氣泡出現在桌子上方
- 完成條件：玩家點擊訂單氣泡（或系統自動展示訂單內容後 3 秒）
- 對話提示：
  - [系統]「客人點餐了！點擊桌子上的訂單氣泡查看他們要什麼。」
  - [阿龍師傅]「三杯雞一份、啤酒兩罐。好，我去炒。」
- UI 高亮：訂單氣泡（綠色閃爍），點擊後展開訂單內容面板
- 跳過條件：3 秒後自動進入下一步（避免玩家不知道要點）

---

### 步驟 5：看阿龍師傅炒菜

- 觸發條件：訂單確認後，廚房出現烹飪進度條
- 完成條件：烹飪進度條跑完，三杯雞完成
- 對話提示：
  - [系統]「廚師正在烹飪。進度條跑完就可以出菜了。」
  - [阿龍師傅]「快炒爐只有一口，同時只能炒一道菜，不要一次叫太多！」
- UI 高亮：快炒爐上方的進度條，白色光暈環繞爐台
- 跳過條件：此步驟等待中可繼續看其他步驟說明（系統彈出「小知識」提示，不強制等待）

---

### 步驟 6：送餐

- 觸發條件：三杯雞完成，出現在出菜台上
- 完成條件：玩家點擊出菜台上的餐點（或點擊「送餐」按鈕）
- 對話提示：
  - [系統]「菜炒好了！點擊出菜台送餐給客人。」
  - [老闆娘]「好，我去端過去！」
- UI 高亮：出菜台，亮橘色邊框；從出菜台到方桌的走道以白色箭頭指引
- 跳過條件：無

---

### 步驟 7：收款，看日結算

- 觸發條件：客人用餐完畢（系統計時 30 秒），離席動畫播放
- 完成條件：玩家點擊收銀台的「收款」圖示（或 5 秒後自動收款）
- 對話提示：
  - [老闆娘]「謝謝光臨，慢走！」
  - [系統]「客人滿意離開！收入 +290 元，名聲 +2。」
- UI 高亮：右上角收入數字閃爍，名聲飄字（+2，金色）從桌子飄起
- 跳過條件：5 秒後自動進入下一步

---

### 步驟 8：組合加成初體驗（Day 1 核心爽感）

- 觸發條件：當日第三組客人結帳，且該桌同時點了「三杯雞」與「台灣啤酒」
- 完成條件：玩家看到組合加成彈出框（無需操作，觀看即可）
- 對話提示：
  - [系統]「組合加成！【台啤絕配】：三杯雞 + 台灣啤酒同桌出現，每桌額外 +40 元、名聲 +1。」
  - [阿龍師傅]「台啤配熱炒，這才是台灣味！記起來，搭對了就有加成。」
- UI 高亮：組合加成彈出框（金色邊框，全螢幕輕微閃爍 1 秒），飄字效果「台啤絕配！+40 元」從桌子位置升起
- 跳過條件：無（此步驟為 Day 1 必看體驗，但彈框 5 秒後自動消失）

備注（程式實作）：系統需確保 Day 1 第三組客人訂單包含三杯雞 + 台啤，為固定腳本，非隨機。

---

### 步驟 9：打烊與日結算

- 觸發條件：Day 1 全部 10 組客人完成服務，或遊戲日 00:00 自動觸發
- 完成條件：玩家看完日結算畫面，點擊「確認」
- 對話提示：
  - [系統]「今日結算：收入 2,200 元 / 食材成本 836 元 / 固定支出 1,650 元 / 淨利 -286 元」
  - [老闆娘]「第一天嘛，正常啦。系統說有補貼，收到了。」
  - [系統]「開業補貼金 +3,000 元。第一天辛苦了！明天可以擴建一下了。」
- UI 高亮：日結算畫面各數字欄位依序亮起（收入 → 成本 → 淨利 → 名聲），補貼金以金色大字跳出
- 跳過條件：無（首次日結算必看完整版）

---

### Day 1 結束狀態

- 資金：約 2,914 元（初始 0 + 第一天淨利 -286 + 補貼 3,000 + 小費約 200）
- 名聲：約 25 點
- 已解鎖操作：開門、查看訂單、手動送餐、收款
- 玩家已體驗：完整一次服務循環 + 一次組合加成

## Day 2：把店弄像個店

### 概述

主題：建造模式教學。
教學目標：讓玩家學會鋪設走道、擺放桌椅、增加座位數量。

設計原則：半引導——系統提示做什麼與在哪裡操作，但不鎖定操作順序，玩家可自由決定先做哪個。

開場設定：
- 日結算結束後，進入 Day 2 的「開店前」時段
- 阿龍師傅主動提起「要不要加張桌子」，引出建造模式

---

### 步驟 1：進入建造模式

- 觸發條件：Day 2 開始，日結算畫面關閉後自動觸發
- 完成條件：玩家點擊右下角的「建造」按鈕，進入施工模式
- 對話提示：
  - [阿龍師傅]「昨天生意還行。就是桌子太少，有人等了沒位置走掉。」
  - [系統]「點右下角的『建造』按鈕，可以擴充你的店面配置。」
- UI 高亮：右下角「建造」按鈕，橘色邊框脈動
- 跳過條件：玩家已進入過建造模式則跳過此步

---

### 步驟 2：鋪設走道格

- 觸發條件：進入建造模式後自動觸發
- 完成條件：玩家鋪設至少 2 格走道（灰色底板）
- 對話提示：
  - [系統]「走道讓員工和客人移動。選『走道』底板，點格子鋪設。走道要從入口連到每張桌子。」
  - [阿龍師傅]「路不通，菜端不到，再好吃也沒用。」
- UI 高亮：底板選擇欄中「走道（灰）」選項高亮；可鋪設的空格以半透明灰色預覽
- 跳過條件：若現有走道已連通入口到所有桌椅，此步驟自動跳過

---

### 步驟 3：擺放第二張桌子

- 觸發條件：走道鋪設完成後觸發
- 完成條件：玩家從設備清單選取「方桌（4 人）」並成功放置在外場區
- 對話提示：
  - [系統]「從設備清單選一張方桌，放在米黃色的外場區。桌子旁要緊鄰走道才能讓客人坐。」
  - [老闆娘]「加桌！這樣今天多接幾桌客人。」
- UI 高亮：左側設備清單中「方桌（4 人）」項目高亮；外場區空格以米黃色閃爍提示可放位置
- 跳過條件：若已有 2 張以上方桌則跳過

---

### 步驟 4：確認路徑連通

- 觸發條件：擺放桌子後，系統進行路徑檢查
- 完成條件：路徑檢查通過（入口到所有桌子有連通走道），或玩家修正後通過
- 對話提示：
  - [系統]（路徑通暢）「路徑連通！客人可以走到這張桌子了。」
  - [系統]（路徑不通）「走道不通！這張桌子客人走不到。虛線箭頭會顯示建議路徑。」
  - [阿龍師傅]「通道堵住的話，客人到不了，那桌等於沒用。」
- UI 高亮：（通暢）新桌子綠色邊框閃一下；（不通）斷開的格子紅色邊框，虛線箭頭顯示建議補路方向
- 跳過條件：路徑本來就通暢則直接進入下一步

---

### 步驟 5：退出建造模式，開門營業

- 觸發條件：路徑確認通暢後觸發
- 完成條件：玩家點擊「完成建造」按鈕，退出施工模式
- 對話提示：
  - [系統]「建造完成。點『完成建造』回到營業模式。」
  - [老闆娘]「好，準備好了，開門！」
- UI 高亮：右下角「完成建造」按鈕
- 跳過條件：無

---

### 步驟 6：觀察新桌子的效果

- 觸發條件：Day 2 營業時段，第一次有客人坐到新增的桌子
- 完成條件：新桌子第一次服務完成，客人滿意離席
- 對話提示：
  - [系統]「新桌子開張！今天可以多接這桌的客人了。」
  - [阿龍師傅]「多一張桌，多一份收入。桌子夠多，客人才不用等。」
- UI 高亮：新桌子位置短暫閃爍（白色光圈）
- 跳過條件：若客人未坐到新桌（例如玩家放錯地方導致路徑不通）則不觸發，等玩家修正後下次營業日再觸發

---

### Day 2 結束狀態

- 資金：Day 1 結餘 + Day 2 收入（預估額外多接 3-5 桌，約增加 600-1,000 元）
- 名聲：約 40-50 點
- 已解鎖操作：建造模式、走道鋪設、桌椅擺放、路徑檢查
- 玩家已體驗：自主改造店面、看到改造帶來的直接效益

## Day 3：招人手、開新菜

### 概述

主題：雇用員工 + 菜單管理。
教學目標：雇用秀蘭姐（S004，外場助手），解鎖第二道菜，體驗組合加成的策略感。

設計原則：只給提示，不鎖操作。玩家可以選擇先雇人或先開菜，系統不強制順序。

備注（角色說明）：
根據 core-design.md 的員工架構，「老闆娘（Lv1）」需名聲 100 才可解鎖，Day 3 名聲約 40-50 尚未達標。
此處「秀蘭姐（S004）」設定為教學專用臨時角色——外場打工仔定位，協助送餐，讓玩家體驗「雇人」的操作流程，但此角色非正式員工系統的「老闆娘」。秀蘭姐是鄰居大姐，幫忙幾天，教學結束後轉為可選擇是否繼續雇用（每日 200 元工資）。

開場設定：
- Day 2 日結算畫面出現後，秀蘭姐主動登場提供幫忙
- 系統同時提示可以解鎖第二道菜

---

### 步驟 1：秀蘭姐登場

- 觸發條件：Day 3 開始（Day 2 日結算關閉後）自動觸發
- 完成條件：玩家看完秀蘭姐的自我介紹（點擊確認）
- 對話提示：
  - [秀蘭姐]「欸，小妹，我是隔壁的秀蘭，聽說妳在請人？我先來幫個忙！」
  - [阿龍師傅]「有人送餐，我專心炒菜，速度快一倍。」
- UI 高亮：地圖上秀蘭姐的角色圖示出現在入口處，周圍橘色光圈
- 跳過條件：無（首次必看）

---

### 步驟 2：雇用秀蘭姐

- 觸發條件：步驟 1 完成後觸發
- 完成條件：玩家進入員工面板並點擊「雇用秀蘭姐（200 元/日）」
- 對話提示：
  - [系統]「點右上角的『員工』按鈕開啟員工面板。雇用員工要先確認每日薪資夠不夠付。」
  - [老闆娘]「行啦，先幫我幾天，算妳 200 一天。」
- UI 高亮：右上角「員工」圖示高亮；員工面板中秀蘭姐的卡片邊框閃爍
- 跳過條件：若玩家當日資金低於 600 元（3 日薪資），系統額外補貼 500 元並提示「開業特別補助」

---

### 步驟 3：觀察秀蘭姐送餐

- 觸發條件：秀蘭姐雇用完成，Day 3 營業時段開始後首次出菜
- 完成條件：秀蘭姐完成第一次自動送餐動作
- 對話提示：
  - [系統]「外場員工會自動從出菜台取菜送到客人桌上，你不需要手動送餐了。」
  - [秀蘭姐]「我來！我來！我眼力好，哪桌的菜我清楚。」
- UI 高亮：秀蘭姐角色圖示上顯示「自動送餐」標籤，移動路徑以虛線箭頭追蹤顯示
- 跳過條件：若雇用後 60 秒內未出菜（廚師還在炒），此步驟暫停等待，出菜後觸發

---

### 步驟 4：開啟菜單管理

- 觸發條件：Day 3 首次進入開店前時段，或玩家自行點擊「菜單」按鈕
- 完成條件：玩家開啟菜單管理面板
- 對話提示：
  - [系統]「菜單可以隨時調整。點右上角的『菜單』按鈕管理今天賣什麼菜。」
  - [阿龍師傅]「現在只有 5 道，太少了。錢夠的話可以解鎖新菜。」
- UI 高亮：右上角「菜單」圖示高亮，點擊後開啟菜單面板
- 跳過條件：玩家已主動開啟菜單面板則此提示不出現

---

### 步驟 5：解鎖「炒空心菜」（第二道熱炒）

- 觸發條件：菜單面板開啟後觸發
- 完成條件：玩家點擊「炒空心菜」的「解鎖」按鈕，支付食譜費（教學期間免費，正常為 3,000 元）
- 對話提示：
  - [系統]「灰色的菜色代表尚未解鎖。點擊查看解鎖條件，再按『解鎖』加入菜單。」
  - [阿龍師傅]「炒空心菜！快，成本低，炒起來又快。配三杯雞賣，老街坊最愛。」
- UI 高亮：菜單面板中「炒空心菜」項目，黃色邊框；「解鎖」按鈕橘色閃爍
- 跳過條件：若炒空心菜已解鎖（例如 Day 2 玩家自行解鎖）則此步驟提示改為「恭喜，你已解鎖炒空心菜！」並跳至步驟 6

備注（數值）：教學期間炒空心菜解鎖費用設為 0 元，目的是讓玩家體驗「解鎖」這個操作動作，正式遊戲中需支付食譜費。

---

### 步驟 6：上架新菜到今日菜單

- 觸發條件：炒空心菜解鎖後觸發
- 完成條件：玩家點擊「加入今日菜單」，炒空心菜出現在今日菜單列表中
- 對話提示：
  - [系統]「解鎖菜色後要手動加入今日菜單，客人才會點。解鎖不等於上架！」
  - [老闆娘]「上架！今天開始賣。」
- UI 高亮：菜單面板中「今日菜單」區塊高亮，「加入今日菜單」按鈕閃爍
- 跳過條件：無

---

### 步驟 7：觀察「在地好味道」組合加成觸發

- 觸發條件：Day 3 營業中，某桌同時點了「炒空心菜 + 皮蛋豆腐 + 三杯雞」（系統腳本確保 Day 3 第五組客人會點這個組合）
- 完成條件：玩家看到「在地好味道」組合加成彈出框
- 對話提示：
  - [系統]「組合加成！【在地好味道】：炒空心菜 + 炒空心菜 + 皮蛋豆腐同日上架，老街坊出現率 +30%，名聲加成翻倍！」
  - [阿龍師傅]「對！這幾道搭在一起，老街坊喜歡。明明便宜，就是有味道。」
- UI 高亮：組合加成彈出框（金色邊框），桌子位置飄字「在地好味道！」
- 跳過條件：若今日菜單未同時包含這三道菜，此步驟跳過（不強制，但這代表玩家沒上架炒空心菜或皮蛋豆腐）

備注：若步驟 7 未觸發（玩家沒上架炒空心菜），Day 3 日結算後補播一段提示：「明天試試看同時上架炒空心菜和皮蛋豆腐，說不定有驚喜！」

---

### 步驟 8：Day 3 日結算 — 教學正式結束

- 觸發條件：Day 3 打烊後，日結算畫面出現
- 完成條件：玩家點擊「確認」，進入正式遊戲
- 對話提示：
  - [系統]「三天教學完成！你學會了：服務流程、建造店面、雇用員工、管理菜單、組合加成。接下來靠你自己了！」
  - [老闆娘]「好，基礎夠了。阿龍，秀蘭，我們好好拼！」
  - [阿龍師傅]「少廢話，快來客人了。」
- UI 高亮：日結算畫面出現「教學完成」橫幅（金色，頂部），下方顯示三天累積數字
- 跳過條件：無（教學結束必看）

### Day 3 結束狀態

- 資金：估計 3,500-5,000 元（依前兩天表現而定，系統確保至少 3,000 元讓玩家有信心繼續）
- 名聲：約 60-80 點
- 員工：阿龍師傅（固定）+ 秀蘭姐（可選擇繼續雇用，每日 200 元）
- 菜單：5 道初始菜色 + 炒空心菜（共 6 道，若玩家在步驟 5 解鎖並上架）
- 已解鎖操作：全部基礎操作（服務循環、建造模式、員工面板、菜單管理）
- 移除限制：教學保護解除，名聲增減回歸正常，客流不再固定腳本

## 附錄：教學對話語氣規範

### 阿龍師傅

- 台式語氣，口頭禪帶點嘴硬：「好啦」、「那當然」、「少廢話」
- 說的是廚師視角的事：火候、備料、菜色搭配、廚房效率
- 不會主動稱讚玩家，只說事實（「多一張桌，多一份收入」而不是「你好棒！」）
- 每句話最多 15 個字

### 老闆娘（玩家角色）

- 實際、直接、有主見
- 做決定的台詞：「好，就這樣」、「行啦」、「開門！」
- 不說廢話，不說太長的句子

### 系統提示

- 無個性，純說明操作方法
- 用「點擊 X 做 Y」句型
- 一則最多 2 句話
- 避免使用「您」，用「你」（親切感）

### 秀蘭姐

- 熱情、自來熟、鄰家大姐氣質
- 說話帶點台語腔調：「欸」、「啦」、「我來我來」
- 只在 Day 3 出現，不過度佔用教學節奏
