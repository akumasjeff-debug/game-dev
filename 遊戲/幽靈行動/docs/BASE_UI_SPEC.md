# 幽靈行動 — 基地場景 UI 設計規格
版本：1.0  |  作者：UI/UX 設計師  |  最後更新：2026-06-21

---

## 設計前提

- 裝置基準：手機直屏 1080 x 1920px（與 HUD_SPEC.md 一致）
- 安全邊距：左右各 40px，頂部 80px，底部 60px
- 所有可點擊元件最小觸控區：120 x 120px（手機安全值）
- 字體規範（延伸自 HUD_SPEC）：
  - 軍事標題：Share Tech Mono 或 Orbitron
  - 中文正文：Noto Sans TC
- 色彩延伸自 GDD 色彩規範：
  - 主背景：#0F160F（比 #1A2B1A 更深，避免基地場景與 HUD 混淆）
  - 面板背景：#1A2B1A（深軍綠）
  - 卡片底色：#202820（略亮軍綠）
  - 重點色：#E8600A（橙）
  - 主線金色：#E8A00A
  - 支線藍色：#4488CC
  - 成功/確認：#44CC44
  - 危險：#CC2222
  - 文字白：#F0F0F0
  - 文字灰：#AAAAAA
  - 邊框暗：#3A4A3A

## 整體版面結構

畫面分為四個垂直區域，從上到下依序排列：

```
┌──────────────────────────────────────────────┐
│  [A] 頂部狀態列           高度 100px           │
│  金幣圖示＋數字   標題文字   設定圖示           │
├──────────────────────────────────────────────┤
│  [B] 任務板              高度 560px           │
│  "MISSION BOARD" 標籤                        │
│  ┌──────────────────────────────────────┐    │
│  │  主線任務卡（橙色左邊框）              │    │
│  └──────────────────────────────────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  支線任務卡 1（藍色左邊框）            │    │
│  └──────────────────────────────────────┘    │
│  ┌──────────────────────────────────────┐    │
│  │  支線任務卡 2（藍色左邊框）            │    │
│  └──────────────────────────────────────┘    │
├──────────────────────────────────────────────┤
│  [C] 陣容選擇區           高度 460px          │
│  "SQUAD" 標籤                                │
│  可用職業：2x3 格 職業按鈕                    │
│  出戰陣容：4 個槽位（橫排）                   │
├──────────────────────────────────────────────┤
│  [D] 出發按鈕             高度 120px          │
│  [       出發執行任務 ▶       ]               │
└──────────────────────────────────────────────┘

總高度使用：100 + 560 + 460 + 120 = 1240px
剩餘 680px 由區塊間距（16px x3）、節標籤（40px）、分隔線（2px）吸收。
實際底部安全邊距 60px，完整畫面 1920px 均可呈現而不需滾動。
```

## 操作流程圖

### 流程一：完整選擇任務並出發

```
[進入基地]
    ↓
[系統判斷] 有離線時間超過 1 分鐘？
    ├── 是 → 顯示離線金幣彈窗
    │          玩家點「收下，繼續」
    │          金幣數字跳動至新值
    │          彈窗退場
    └── 否 → 跳過
    ↓
[基地畫面顯示]
玩家看到頂部金幣 + 任務板 + 陣容區
    ↓
玩家點擊任務卡片的「選擇」按鈕
    ↓
[視覺] 按鈕縮小回彈 → 按鈕變橙色「已選取 ✓」→ 卡片邊框亮橙
    ↓
玩家點擊職業按鈕（2x3 Grid）加入陣容
    ↓
[視覺] 職業按鈕邊框亮起、右下角 ✓ → 對應陣容槽位填入職業圖示
[每加入一人] 出發按鈕提示文字更新「已選 X / 4」
    ↓
[陣容滿 4 人]
[視覺] 出發按鈕從灰色禁用 → 橙色可點擊，發光脈衝啟動
    ↓
玩家點擊出發按鈕
    ↓
[視覺] 按鈕縮小 98%（20ms）
[音效] 出發音效
[邏輯] 存檔 → 清除上局殘留 → 切換至戰場場景
```

### 流程二：調整陣容（移除已選角色）

```
玩家點擊職業按鈕（已在陣容中）
    ↓
[視覺] 按鈕淡出 150ms → 更新為未選狀態
[視覺] 對應陣容槽位改為「空」（灰色虛線框）
[出發按鈕] 若陣容不足 4，重新進入禁用狀態
    ↓
玩家點擊陣容槽位（另一種移除方式）
    ↓
[視覺] 槽位邊框瞬間變紅（0.2 秒）→ 清空
[視覺] 對應職業按鈕恢復未選狀態
```

### 流程三：誤操作處理

```
情況 A：點擊出發按鈕，陣容不足或未選任務
    ↓
[視覺] 按鈕左右震動（-4px → +4px → 0，120ms）
[文字] 按鈕上方紅色提示說明缺少什麼
[音效] 短促錯誤音（低沉的「嗶」聲）
    ↓
玩家修正後，提示文字自動消失，出發按鈕進入可點狀態

情況 B：陣容已滿時點擊新職業
    ↓
[視覺] 職業按鈕左右震動
[Toast] 螢幕中央上方出現 Toast（HUD_SPEC 規格）：
        「陣容已滿（4/4）  請先移除一名隊員」
        持續 2 秒後淡出
```

## UX 問題清單

| 編號 | 問題描述 | 嚴重度 | 建議對策 |
|------|---------|--------|---------|
| UX-B-01 | 任務卡說明文字截斷，玩家可能想看完整描述 | 中 | 點擊卡片主體（非按鈕）可展開卡片至 320px 高，顯示完整說明；再次點擊收合。MVP 可先跳過此功能。 |
| UX-B-02 | 獎勵圖示太小（18px），手機上看不清楚是什麼道具 | 中 | 獎勵文字旁顯示完整文字說明，不純靠圖示傳達 |
| UX-B-03 | 玩家不知道可以選多個支線任務還是只能選一個 | 高 | 任務板標題副文字補充「每次只能選擇 1 個任務出發」，選擇後其他任務的按鈕需明確變灰（不消失） |
| UX-C-01 | 2x3 職業按鈕格，玩家不知道點了是「加入」還是「選擇」 | 高 | 未選狀態按鈕顯示「點選加入」或「+」圖示，選中後顯示「✓」，移除需點陣容槽，避免混淆 |
| UX-C-02 | 陣容已滿時新職業按鈕視覺變灰，玩家可能以為是 bug | 中 | 變灰按鈕同時顯示小鎖圖示，並在 hover/長按時出現 tooltip「陣容已滿，移除後可替換」 |
| UX-C-03 | 4 個陣容槽沒有排序說明，玩家不知道槽位順序有無意義 | 低 | 槽位上方加「出戰順序」標籤（用 1-2-3-4 編號），說明槽位=隊列位置（由程式依職業自動調整） |
| UX-D-01 | 出發按鈕過大，可能在滾動頁面時誤觸 | 低 | 出發按鈕設計為需要稍微用力停留點擊（即按住 0.2 秒才觸發），防止手指滑過時誤觸。MVP 先做瞬按。 |
| UX-E-01 | 離線彈窗強制擋住畫面，若玩家只是快速開啟確認金幣會覺得惱人 | 低 | 確認按鈕尺寸夠大（80px 高），讓玩家快速點擊即可關閉，不增加額外步驟 |
| UX-E-02 | 若玩家剛離開幾秒就重新進入，顯示「0 金幣」獎勵彈窗很奇怪 | 中 | 設定最低顯示門檻：離線時間 > 1 分鐘且金幣 > 0 才顯示（base.gd 的 `_show_offline_reward` 已實作此邏輯） |

## 與 HUD_SPEC.md 的銜接說明

本規格文件設計的基地 UI 與 HUD_SPEC.md 共用以下視覺語言：

- 職業圖示圓圈顏色完全一致（確保玩家在基地選人 → 進入任務後，卡片上看到的是同一個職業色）
- 字體規範相同（Share Tech Mono 標籤 / Noto Sans TC 正文）
- Toast 飄字規格沿用 HUD_SPEC 的 Toast 元件規格
- 按鈕按下縮放動畫幅度一致（縮小至 96-98%，20ms 回彈）

基地場景的橙色重點（#E8600A）與任務場景的 HUD 重點色相同，確保視覺主軸貫通整個遊戲。
