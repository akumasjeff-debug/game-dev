## 二、問題清單

### 優先修改（影響玩家體驗 / 邏輯錯誤）

**P1-01｜decision_trigger.gd 的 fallback 文字與 events.json 不同步**
- 位置：`decision_trigger.gd` 第 28-43 行（`BASE_DECISION_DATA`）
- 現況：
  - fallback 選項標籤：`"直衝進入"`、`"靜悄悄進入"`、`"投擲炸彈"`
  - events.json 實際選項標籤：`"直衝突入"`、`"靜悄進入"`、`"炸彈前置"`
- 問題：若 events.json 讀取失敗，玩家看到的選項文字與 JSON 裡的不一樣，術語分裂
- 建議修改：
  ```
  "直衝進入" → "直衝突入"
  "靜悄悄進入" → "靜悄進入"
  "投擲炸彈" → "炸彈前置"
  ```
  補給欄位同理：
  ```
  "全體補血" → "取血包"
  "補充炸彈" → "取炸彈"
  "取得抽卡券" → "取抽卡券"
  ```

**P1-02｜decision_trigger.gd fallback 標題語氣不符軍事風格**
- 位置：`decision_trigger.gd` 第 26-27、36-37 行
- 現況：
  - `"title": "前方發現房間"` → 語法不通順（「發現房間」像機器翻譯）
  - `"description": "偵察顯示房間內有敵人，如何進入？"` → 語氣偏書面，缺乏即時張力
  - `"description": "補給箱完好無損，選擇補給項目："` → 語氣過於平鋪
- 建議修改：
  ```
  "前方發現房間" → "前方房間"
  "偵察顯示房間內有敵人，如何進入？" → "偵測到敵方移動，如何進入？"
  "補給箱完好無損，選擇補給項目：" → "醫療兵：補給箱完好，拿什麼？"
  ```

**P1-03｜hud.gd 任務失敗文字「全員陣亡」對我方使用不當**
- 位置：`hud.gd` 第 402 行
- 現況：`"小隊全員陣亡，任務宣告失敗。"`
- 問題：「陣亡」通常用於敵方戰死或正式軍事公報，我方隊員失去戰力更適合用「倒下」或「覆沒」
- 建議修改：`"小隊全員倒下，任務宣告失敗。"`

**P1-04｜main.gd 岔路選項文字格式與 events.json 不一致**
- 位置：`main.gd` 第 285-287 行
- 現況：
  ```
  "左路（困難）" / "右路（簡單）" / "探索未知路線"
  ```
- events.json 格式：`"走左路"` / `"走右路"`，難度說明放在 `"desc"` 欄位
- 問題：括號標籤把選項資訊壓在 label 裡，和正式資料格式不符；且「（困難）」、「（簡單）」直接洩露結果，破壞策略感
- 建議修改：
  ```
  "走左路" / "走右路" / "走未知路"
  （難度資訊移入 desc 欄位）
  ```

**P1-05｜base.gd 按鈕文字「出發執行任務」超出建議長度**
- 位置：`base.gd` 第 259 行
- 現況：`"出發執行任務"` — 6 字（剛好在邊界，但手機橫向空間有限）
- 問題：中文按鈕建議 6 字以內；加上按鈕邊距與字體，實際顯示可能截斷
- 建議修改：`"出發"` 或 `"執行任務"`（依按鈕實際寬度二選一）

### 次要修改（一致性問題，不緊急）

**P2-01｜base.gd 任務難度直接顯示 JSON 英文值**
- 位置：`base.gd` 第 162 行
- 現況：`diff_lbl.text = "難度：" + mission["difficulty"]`
- 問題：difficulty 欄位值為 `"intro"` / `"normal"` / `"hard"`，直接顯示英文給玩家看
- 建議：加對照表轉換
  ```gdscript
  var diff_map = {"intro": "入門", "normal": "普通", "hard": "困難"}
  diff_lbl.text = "難度：" + diff_map.get(mission["difficulty"], mission["difficulty"])
  ```

**P2-02｜enemy.gd 敵人名稱與 missions.json 的 enemy_types 定義不符**
- 位置：`enemy.gd` 第 50 行、55 行
- 現況：
  - `enemy_name = "普通兵"`（fallback）
  - `enemy_name = "精英"`（fallback）
- missions.json 定義：
  - `"guard_normal"` → `"普通警衛"`
  - `"elite_guard"` → `"精英保全"`
- 問題：fallback 名稱和資料表不一致，除錯時難以識別
- 建議修改：
  ```
  "普通兵" → "普通警衛"
  "精英" → "精英保全"
  ```

**P2-03｜character.gd 預設值「大招」過於通用**
- 位置：`character.gd` 第 17 行
- 現況：`@export var ultimate_name: String = "大招"`
- 問題：預設值若出現在 UI 上，玩家會看到「大招」而非實際技能名稱
- 建議修改：改為佔位標記 `"（未命名）"` 或保持空字串

**P2-04｜tutorial_manager.gd 教學語氣略偏輕鬆**
- 位置：`tutorial_manager.gd` 第 189、197、228 行
- 現況：
  - `"小隊自動前進，你不用操控移動"` — 「你不用」太口語
  - `"輪到你了，選一個進入方式"` — 「輪到你了」像桌遊提示
  - `"你已掌握核心指揮技巧，繼續完成任務！"` — 尚可，語氣略軟
- 教學文字的語氣可以稍指揮官風，但也需要新手友善，保持平衡
- 建議修改（不強制，可依整體調性決定）：
  - `"小隊自動前進，你不用操控移動"` → `"小隊自動推進，毋需手動操控"`
  - `"輪到你了，選一個進入方式"` → `"指揮官決策中——選擇進入方式"`

**P2-05｜hud.gd 進度標籤文字過短缺乏脈絡**
- 位置：`hud.gd` 第 341 行
- 現況：`progress_label.text = "進度 " + str(int(ratio * 100)) + "%"`
- 建議：改為更有軍事感的格式，例如 `"推進 42%"` 或 `"任務進度 42%"`

**P2-06｜base.gd 任務板標題過於樸素**
- 位置：`base.gd` 第 105 行
- 現況：`section_lbl.text = "任務板"`
- 建議：可改為 `"任務指派"` 或維持原樣（低優先度）
