# 幽靈行動 — 新手教學設計文件
版本：1.0｜建立：2026-06-21

---

## 設計目標

在 3 分鐘內讓玩家理解四件事：
1. 小隊自動走，你不用管移動
2. 畫面停住時輪到你選擇
3. 底部卡片可以點——這是你的主動工具
4. 出發前要選對人

教完即止。進階機制（升等、隱藏事件、岔路策略）讓玩家自己探索。

---

## 教學關卡設計規格

### 場景規格

- 地圖：2 個房間 + 1 個岔路 + 終點
- 敵人：數量比正常少 30%，HP 降低 40%（教學專屬數值）
- 預設陣容：盾兵、突擊手、醫療兵、爆破手（已選好，玩家不需決定）
- 大招 CD：全部縮短至 15 秒（教學專用，確保玩家能在教學期間觸發）
- 時長預估：90–120 秒

## 教學步驟腳本

---

### 步驟 0：組隊畫面引導（遊戲首次啟動）

```
觸發條件   ：第一次進入「選擇隊員」畫面
畫面提示   ：「選 4 人出發」
高亮 / 遮罩：遮住所有按鈕；僅高亮 6 個角色卡片區域
遮罩透明度 ：非高亮區域 60% 黑色遮罩
提示位置   ：提示文字浮現於角色卡片區上方，白色字，橙色底框
完成條件   ：玩家點擊任意 4 張卡片（完成選擇）
→ 下一步    ：步驟 1
備註       ：教學關卡不顯示職業詳細說明——留到玩家第一次自由組隊時再出現
             tooltip（見「職業說明 Tooltip」章節）
```

---

### 步驟 1：讓玩家「看」自動推進

```
觸發條件   ：進入教學關卡，小隊開始移動的第 1 秒
畫面提示   ：「小隊自動前進，你不用操控移動」
高亮 / 遮罩：無遮罩；提示文字疊在畫面下方 HUD 上方
             一條橙色橫線從左到右掃過，指向前進方向（動畫一次，約 1 秒）
完成條件   ：計時 4 秒後自動消失，無需玩家操作
→ 下一步    ：步驟 2
設計意圖   ：讓玩家先「觀看」一小段自動推進，體感建立後才繼續教
```

---

### 步驟 2：第一個決策點——教「畫面會暫停」

```
觸發條件   ：小隊抵達第一個房間門口（決策點觸發、畫面暫停的瞬間）
畫面提示   ：「輪到你了，選一個進入方式」
高亮 / 遮罩：決策選項面板正常顯示；遮住面板以外所有區域（60% 遮罩）
             三個選項各自有清晰的圖示（門的圖示、腳步圖示、炸彈圖示）
             箭頭從提示文字指向選項面板
完成條件   ：玩家點擊任意一個選項
→ 下一步    ：步驟 3
設計意圖   ：不告訴玩家「哪個選項最好」——讓他自己做決定，感受決策的主控感
             三個選項都有效，不設陷阱
```

---

### 步驟 3：選擇執行後的回饋確認

```
觸發條件   ：步驟 2 的選項執行完畢，小隊進入第一個房間並開始交戰
畫面提示   ：「選擇執行了！小隊繼續前進」
高亮 / 遮罩：無遮罩；提示文字顯示 2 秒後淡出
完成條件   ：計時 2 秒後自動消失
→ 下一步    ：步驟 4
設計意圖   ：確認「選擇 → 執行」的因果閉環，讓玩家知道自己的決定有效
```

---

### 步驟 4：大招教學——教「底部卡片可以點」

```
觸發條件   ：第一個房間交戰進行中（敵人出現後 3 秒）
畫面提示   ：「點擊角色卡片，立刻發動大招！」
高亮 / 遮罩：遮住畫面其他所有區域（70% 遮罩）
             僅高亮底部「突擊手」卡片（橙色邊框發光脈衝動畫）
             箭頭從提示文字指向突擊手卡片
完成條件   ：玩家點擊突擊手卡片，觸發「火力全開」大招
→ 下一步    ：步驟 5
設計意圖   ：指定突擊手，因為他的大招（攻擊力提升）視覺回饋最即時且明顯
             玩家能直接看到敵人更快倒下的效果
```

---

### 步驟 5：大招 CD 說明——教「CD 期間卡片變暗」

```
觸發條件   ：步驟 4 的大招觸發後 0.5 秒（卡片進入 CD 變暗狀態）
畫面提示   ：「冷卻中，稍後可再次使用」
高亮 / 遮罩：高亮突擊手卡片（CD 計時視覺）；其他卡片正常顯示
             卡片上有順時針的 CD 計時圓圈覆蓋（Godot ProgressBar 或 TextureProgressBar）
完成條件   ：計時 3 秒後自動消失（不需等 CD 結束）
→ 下一步    ：步驟 6
設計意圖   ：只說「變暗 = 等等」，不細解 CD 秒數——玩家自己觀察就能理解節奏
```

---

### 步驟 6：第二個決策點——讓玩家獨立操作

```
觸發條件   ：小隊抵達岔路決策點（第二個決策點）
畫面提示   ：（無提示文字）
高亮 / 遮罩：無遮罩；決策面板正常顯示
完成條件   ：玩家點擊任意一個岔路選項
→ 下一步    ：步驟 7
設計意圖   ：「不幫忙」是最好的測試——確認玩家已能獨立應對決策點
             若玩家 10 秒未操作，才出現淡色提示「選一條路繼續前進」（不高亮、不遮罩）
```

---

### 步驟 7：教學結束確認

```
觸發條件   ：步驟 6 完成後
畫面提示   ：「你已掌握核心指揮技巧，繼續完成任務！」
高亮 / 遮罩：無遮罩；提示文字於畫面正中央顯示 3 秒，背景半透明黑色卡片
完成條件   ：計時 3 秒後自動消失，或玩家點擊任意位置跳過
→ 下一步    ：遊戲繼續，後續無引導（玩家自由完成教學關卡）
設計意圖   ：明確宣告「你會了」，給予成就感，然後放手
```

## CD 完成提示動畫規格

### 觸發時機

大招 CD 倒數結束的瞬間。

### 視覺效果（依優先順序，程式實作擇一或組合）

```
效果 1 — 卡片脈衝發光
  動作：卡片邊框從暗色閃爍至橙色（#E8600A），脈衝 2 次
  時長：每次脈衝 0.3 秒，共 0.6 秒
  停止：脈衝結束後維持橙色邊框（等待玩家點擊）

效果 2 — 彈跳縮放
  動作：卡片整體縮放 1.0 → 1.15 → 1.0（彈跳感）
  時長：0.3 秒
  Easing：ease_out_back

效果 3 — 文字標籤（補充，非強制）
  內容：「可用！」
  字型：粗體，橙色
  位置：卡片左上角或正上方
  顯示時長：直到玩家點擊使用為止，或 5 秒後自動消失

效果 4 — 音效（必須）
  音效：金屬「叮」聲，短促（0.2 秒內），音量中等
  設計意圖：手機玩家視線不一定在底部，聲音提示比視覺更可靠
```

### 教學關卡 vs 正式關卡

- 教學關卡：效果 1 + 效果 3 + 效果 4（初次接觸，視覺提示要強）
- 正式關卡：效果 1 + 效果 4（效果 2 和 3 關掉，避免視覺噪音）
- 效果 3 的文字在玩家完成教學後永久關閉（不重複出現）

## 職業說明 Tooltip

### 觸發時機

第一次自由組隊時（教學關卡結束後、首次進入正式組隊畫面），玩家長按或懸停（PC）任意角色卡片 0.5 秒後顯示。

### 設計規則

- 每個職業限 2 行，合計不超過 30 字
- 第 1 行：定位（做什麼）
- 第 2 行：大招是什麼

### 六職業 Tooltip 文字

```
盾兵
  站最前排承受傷害，讓隊友安全輸出
  大招：全隊減傷 50%，持續 5 秒

突擊手
  全速連射，清掉眼前所有威脅
  大招：全隊攻擊力 +60%，持續 8 秒

爆破手
  炸桶、破掩體，一炸清空整個房間
  大招：房間所有敵人立即扣 70% HP

醫療兵
  自動回血，讓隊伍撐過最艱難的戰鬥
  大招：全隊立即恢復 30% 最大 HP

狙擊手
  從最後排點殺最危險的敵人
  大招：指定一個敵人，直接秒殺

偵察手
  提早揭露前方情況，讓你搶先準備
  大招：封鎖所有敵人攻擊能力 5 秒
```

### Tooltip UI 規格

```
樣式    ：深色圓角背景（#1A2B1A，80% 不透明），白色文字
位置    ：角色卡片正上方，若上方空間不足則顯示於右側
尺寸    ：寬度固定 180px（手機），高度自動
消失方式：放開手指 / 移開滑鼠後 0.3 秒淡出
```

## 跳過教學方案

### 建議方案：提供跳過但設門檻

```
入口位置  ：步驟 0（組隊畫面）右上角小字按鈕「跳過教學」
字型大小  ：比主要 UI 小 20%，顏色偏暗（不主動引導，但存在）

點擊後行為：
  彈出一行確認文字「跳過後直接進入正式任務，確定？」
  兩個選項：「確定跳過」/ 「繼續教學」
  選「確定跳過」→ 直接進入基地畫面，教學關卡標記為完成

不建議做法 X：
  不要「第一個按鈕就是大大的跳過」——首次玩家不知道自己需不需要教學
  不要完全移除跳過——回流玩家（刪了重裝）和已熟悉類型的玩家需要這個選項
```

### 跳過後補救機制

玩家跳過教學後，第一次觸發大招 CD 完成時，仍顯示效果 1 + 效果 4（CD 提示動畫）。其他步驟不重複。

## 教學結束後的自然銜接

教學關卡完成後進入結算畫面，加入一句引導文字：

```
結算畫面文字（新增於獎勵清單下方）：
「帶不同的人，看到不同的機會。試試換一套陣容出發。」

設計意圖：不明說「去抽卡」，但暗示組隊策略有更多探索空間，
          引導玩家自然進入基地的「角色管理」功能。
```

## 實作備註（給程式設計師）

- 遮罩建議用 Godot `ColorRect` 加 `mouse_filter = MOUSE_FILTER_STOP` 實作，只在高亮區域挖空讓事件穿透
- CD 計時圓圈建議用 `TextureProgressBar`（Godot 4 內建），設定 `fill_mode = FILL_CLOCKWISE`
- 步驟推進器（TutorialManager）用 Signal Bus 監聽「大招觸發」、「決策點完成」等事件，不要在各系統內硬寫 tutorial 邏輯
- 教學狀態存入 ConfigFile，key: `tutorial/completed`，跳過時同樣設 true
- 步驟 6 的 10 秒計時器：若決策點 UI 已顯示，TutorialManager 啟動 Timer，到時才顯示淡色提示
