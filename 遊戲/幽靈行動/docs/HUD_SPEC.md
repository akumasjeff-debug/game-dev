# 幽靈行動 — HUD 設計規格文件
版本：1.0｜作者：UI/UX 設計師｜最後更新：2026-06-21

---

## 設計前提

- 裝置基準：手機直屏 1080 x 1920px
- 安全邊距（Safe Zone）：左右各 40px，頂部 80px（避開系統狀態列），底部 60px（避開手勢導航區）
- 所有可點擊元件最小觸控區：120 x 120px
- 層疊順序（從下到上）：遊戲場景 → 常駐 HUD → 決策面板遮罩 → 決策面板 → Toast 飄字
- 字體：等寬軍事風格（建議 Share Tech Mono 或 Orbitron），正文用 Noto Sans TC

## UX 問題清單（潛在困惑點）

| 編號 | 問題描述 | 嚴重度 | 建議對策 |
|------|---------|--------|---------|
| UX-01 | 玩家不知道卡片可以點擊（大招系統） | 高 | 新手教學第一個大招強制高亮 + 手指圖示引導 |
| UX-02 | CD 數字太小，在手機上看不清楚 | 高 | CD 倒數使用 36px 大字，剩餘 5 秒額外放大 |
| UX-03 | 四張卡片橫排，手機上每張只有 236px 寬，低視力玩家可能辨認困難 | 中 | 職業圖示圓圈顏色應足夠對比，不依賴圖形細節辨認 |
| UX-04 | 鎖定選項讓玩家誤以為遊戲壞掉 | 中 | 鎖定選項顯示解鎖條件，並可點擊觸發說明 Toast |
| UX-05 | 決策面板在玩家手指習慣位置（畫面中下方）反覆出現，可能誤觸底部卡片 | 中 | 決策面板進場時，底部卡片列同步變暗（無法點擊），退場後恢復 |
| UX-06 | 失敗結算的「關鍵提示」如果顯示的條件太難達成，會讓玩家感覺挫敗 | 低 | 只顯示「2 個等級以內可達成」的提示，距離太遠則不顯示 |
| UX-07 | 頂部進度條資訊太少，玩家不知道任務還有多長 | 低 | 進度條下方可顯示「剩餘房間數」（如「還有 3 個房間」），可選功能 |
| UX-08 | 醫療包/炸彈消耗品計數位置（緊貼進度條）在斜角俯視場景中可能和場景物件視覺混淆 | 低 | 計數背景加半透明色塊，確保和場景對比 |

## 操作流程：玩家點擊大招

```
[狀態]：大招就緒，卡片發光

玩家點擊卡片
    ↓
[視覺]：卡片縮小 97%（20ms）→ 橙色高亮閃光（50ms）→ 回彈
[音效]：大招觸發音效
[邏輯]：大招效果立即生效（不暫停遊戲）
    ↓
[視覺]：卡片進入 CD 冷卻狀態（變暗 + 大字倒數出現）
[視覺]：戰場出現大招特效（由視覺動態師另行規格）
    ↓
[倒數中]：每秒更新 CD 數字
[剩餘 5 秒]：數字變橙色 + 放大
    ↓
[CD 結束]：遮罩消失，卡片重新發光，脈衝動畫啟動
[音效]：短促提示音「大招可用」
```

## 操作流程：決策點觸發

```
[遊戲中]：小隊自動推進

系統觸發決策點
    ↓
[音效]：「停止」決策音效（金屬電子音）
[邏輯]：遊戲邏輯暫停
[視覺]：全螢幕半透明遮罩出現（200ms 淡入）
[視覺]：決策面板從中央展開（200ms easeOut）
[視覺]：底部卡片列同步變暗（不可點擊）
    ↓
[玩家看到]：事件標題 + 情境描述 + 選項按鈕
    ↓
玩家點擊一個選項
    ↓
[視覺]：按鈕按下態（縮小 + 高亮，20ms）
[視覺]：面板退場動畫（150ms 縮合）
[視覺]：遮罩淡出（150ms）
[邏輯]：遊戲邏輯恢復
[視覺]：Toast 飄字說明執行結果（如「炸彈清場！敵人受到重傷」）
[視覺]：底部卡片列恢復正常
    ↓
[遊戲繼續]：小隊恢復推進
```
